# Описание проекта

daylist - это offline-first приложение для управления личными задачами, идеями и финансами

## Основные функциональные возможности

- список дел
- заметки
- дневник доходов и расходов

## Технологии

### Лендинг (`apps/landing`, `daylist.com`)

- Astro — статическая генерация, SEO, быстрая загрузка
- Tailwind CSS, TypeScript

### Приложение (`apps/app`, `app.daylist.com`)

- Vite + React — чистый SPA
- Tailwind CSS, TypeScript
- shadcn/ui, Radix UI
- Storybook
- IndexedDB — для офлайн-режима

### Бэкенд

- Nest.js,
- postgresql,
- typeorm

### Инфраструктура

- docker / docker-compose — локальное окружение
- Makefile — команды для запуска миграций, очистки базы и прочих операций
- github actions — CI/CD
- деплой: VPS, БД возможно у провайдера — решение позже

## Аутентификация

- email / пароль
- OAuth через Google

## Синхронизация

- синхронизация доступна только при наличии аккаунта и интернет-соединения
- конфликты разрешаются по полю updatedAt — побеждает запись с более поздней датой изменения (last write wins)

## Цели проекта

- создать удобное и функциональное приложение для управления личными задачами, идеями и финансами, которое может
  работать без необходимости регистрации и авторизации в оффлайн-режиме, сохраняя данные на устройстве пользователя
- обеспечить синхронизацию данных между устройствами при наличии интернет-соединения и зарегистрированного аккаунта

## Требования к интерфейсу

### Профиль

- отображается email
- кнопка выхода из аккаунта
- больше ничего до явной необходимости

### Навигация

- мобильные (до 768px): нижний таббар
- десктоп (768px и выше): боковое меню (sidebar)

### Общие

- все сущности (задачи, заметки, записи о доходах/расходах, категории) имеют обязательные поля: id (uuid), createdAt, updatedAt
- адаптивный дизайн для различных устройств 320px, 768px, 1024px, 1600px
- максимальная ширина экрана - 1600px. При этой ширине и больше, контент центрирован, а по краям экрана остаются поля
- одна цветовая схема - что-то среднее между светлой и темной темой, спокойные тона с минимальной нагрузкой на зрение

### Список дел

- интуитивно понятный и простой в использовании интерфейс, drag-and-drop, возможность вкладывать подзадачи в задачи
  перетаскиванием
- организация по пользовательским категориям; одна категория создаётся по умолчанию — пользователь может сразу начать работу
- перемещение между категориями доступно только для задач верхнего уровня вложенности; при перемещении все подзадачи следуют за родительской задачей

### Заметки

- простой текстовый редактор без форматирования (на первом этапе)
- заметка может быть привязана к задаче, к записи о доходе/расходе, или существовать самостоятельно
- организация по пользовательским категориям (создаются и управляются пользователем); одна категория создаётся по умолчанию
- сортировка и поиск — вне первого этапа
- приоритет первого этапа: минимализм и производительность

### Дневник расходов и доходов

- запись содержит: сумму, тип (доход/расход), описание, дату, валюту
- тип записи (доход/расход) указывается явно
- организация по пользовательским категориям (создаются и управляются пользователем); одна категория создаётся по умолчанию
- на первом этапе — только RUB, без возможности выбора; архитектура должна допускать добавление других валют в будущем

### Аналитика по финансам

- баланс за выбранный период (доходы минус расходы)
- разбивка по категориям
- графики динамики доходов и расходов
- сравнение периодов (например, текущий месяц vs предыдущий)
- средние значения — средний расход в день и в месяц
- топ категорий по объёму трат
- крупнейшие транзакции за период
- процент сбережений (доля дохода, которая не была потрачена)
